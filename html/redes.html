CLASES DE DIRECCIONES IP

La razon por la cual la direccion ip llega a destino es internet es que cada red conectada a internet tienen un numero de red que es unico y exclusivo, para garantizar que sea exclusivo una organizacion internacional de informacion de red(INTERNIC), asigna bloques de direcciones ip a las empresas basandose en sus tamaños de sus redes


Cada direccion IP consta de dos partes: el numero de red y el numero de host. El numero de red identifica la red que forma parte del dispostivo. El numero de host identifica la direccion del dispositivos en dicha red.


Internic asigna 3 clases de direcciones IP, las direcciones de IP clase A para identidades gubernamentales, las direcciones IP de clase B para empresas medianas y las direcciones IP clase C para todo el resto.

Cuando las direcciones IP de clase Ase escriben en formato binario el primer campo siempre es 0. Cuando las direcciones IP de clase B se escriben en formato binario los dos primeros bits son siempre 1 y 0. Cuando las IP de clase C se escriben en formato binario los primeros 3 bits son siempre 1,1 y 0


CLASES DE REDES RESERVADAS

Como vimos 3 clases de redes que pueden ser asignada por internic, aunque existen 5 clases de direcciones de red. Los otros dos se usan para multicast la D, y para investigacion la clase E.


RED CLASE A=RED-HOST-HOST-HOST


RED CLASE B=RED-RED-HOST-HOST


RED CLASE C=RED-RED-RED-HOST







Ademas las direcciones IP reservadas, queda reservada cualquier direccion IP que tenga solo numeros 0 o solo 1 en la parte de direcciones que refiera al host.


Hay ocasiones en las que se hace necesario referirse a todos los dispositivos de la red, por esta razon se ha diseñado un metodo abreviada para referirse a la red.


En los sistemas de direccionamiento IP, cualquier direccion que termina en 0 binario en la parte referente al host queda reservada por convension para la direcciones red.


Tambien queda determinado por convension que el campo referente al host esta compuesto por una serie de interrumpidos 1 binarios se utilizara para direccion de broadcast.


Se produce un broadcast cuando un equipo de origen envia datos a todos los dispositivos de la red y para que todos los dispositivos presten atencion, se utiliza una direccion IP que todos puedan reconocer y captar como vimos una serie interrumpidos 1 binarios en el host.


Para la transferencia de datos en internet, una red percibe a otra como una red integra y no tiene conocimiento de su estructura interna, estoy ayuda a que las tablas de enrutamiento sea pequeñas.


Sin embargo las redes internas pueden dividirse en redes mas pequeñas especialmente las de gran envergadura denominada subredes, esto le va a permitir al administrador mejorar la administracion y administrar flixibilidad adicional.


DIRECCIONAMIENTO DE SUBRED

Tal como sucede con la parte de numero de host de las direcciones es A,B y C, las direcciones de subred se asignan de forma local, el encargado de hacer esto es el administrador de red. Al igual que otras direcciones IP, la direccion de red es unica. Al utilizar sus redes no se produce ningun cambio en la que la forma en el que el exterior percibe la red pero si crean estructura adicional dentro de la organizacion.

255.255.255.0

R - R- R- 0



Los administradores de red reciben el tamaño de las subredes basandose en las necesidades de la organizacion.


Una direccion de subred incluye un numero de red, un numero de subred dentro de la red y un numero de host dentro de la subred.


Para crear una direccion de subred, el administrador pide prestado bits dentro del campo de host y los designa como campo de subred. Se pueden pedir prestado cualquier cantidad de bits siempre y cuando queden 2 bits disponibles.


Para crear una direccion de subred, el administrador de red pide prestado bits del campo de host y los designa como campo de subred.


Cada vez que se pide bit en el campo de host queda un bits menos que se puede utilizar para el numero de host, por ejemplo: en una red de clase c con 3 bits prestados.
